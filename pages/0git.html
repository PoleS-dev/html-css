Guide Git et GitHub

1. Installation de Git

Installez Git selon votre système :

  • Windows : Télécharger et installer depuis git-scm.com
  • MacOS : Utiliser la commande brew install git
  • Linux : Utiliser la commande sudo apt install git (Ubuntu/Debian) ou sudo dnf install git (Fedora)

2. Création d’un dépôt Git

git init

Cette commande initialise un dépôt Git dans le dossier actuel.

3. Ajouter et valider les modifications

git add .
git commit -m "Premier commit"

Ajoute tous les fichiers et enregistre un commit avec un message.

4. Connexion à GitHub et création d’un dépôt distant

git remote add origin https://github.com/utilisateur/mon-depot.git

Associe votre dépôt local à GitHub.

5. Envoi du code sur GitHub

git push -u origin main

6. Collaboration en groupe

  • Cloner un dépôt : 
    git clone https://github.com/utilisateur/mon-depot.git
  • Créer une nouvelle branche : 
    git checkout -b ma-branche
  • Mettre à jour son dépôt local : 
    git pull origin main
  • Envoyer ses modifications : 
    git push origin ma-branche
  • Créer une pull request sur GitHub

7. Travailler à plusieurs sur un même projet

Voici les étapes pour collaborer efficacement :

  1. Un membre crée un dépôt sur GitHub et ajoute les autres comme collaborateurs.
  2. Chaque membre clone le dépôt avec git clone.
  3. Avant de commencer, synchroniser le dépôt local avec git pull origin main.
  4. Créer une branche pour travailler sur une nouvelle fonctionnalité : git checkout -b nouvelle-fonction.
  5. Faire des modifications, puis les enregistrer avec git add . et git commit -m "Ajout de nouvelle fonctionnalité".
  6. Envoyer la branche sur GitHub : git push origin nouvelle-fonction.
  7. Créer une pull request pour demander la fusion de la branche dans main.
  8. Une revue de code est effectuée avant la fusion.
  9. Une fois la pull request fusionnée, chaque membre doit synchroniser son dépôt local avec git pull origin main pour récupérer les dernières modifications.

8. Suppression d’un dépôt

Local :

rm -rf .git

Sur GitHub : Aller dans les paramètres du dépôt et cliquer sur "Delete this repository".

9. Commandes essentielles

  • git status → Vérifier l’état des fichiers
  • git log → Voir l’historique des commits
  • git diff → Voir les modifications non validées
  • ls → Lister les fichiers du répertoire
  • cd nom_du_dossier → Changer de répertoire
  • pwd → Afficher le chemin du répertoire actuel
  • mkdir nom_du_dossier → Créer un nouveau dossier
  • rm -rf nom_du_dossier → Supprimer un dossier et son contenu
  • touch fichier.txt → Créer un fichier vide
  • cat fichier.txt → Afficher le contenu d'un fichier
  • nano fichier.txt → Éditer un fichier avec nano
